# pvNXT HOMEPAGE PLAN v2 — Fresh Perspective

> **Scenario:** I have only the `final_terranxt_index.html` (design reference) + `pvNXT_overview.md` (content). No previous plans. No Next.js. Fresh thinking.

---

## 🧠 FIRST PRINCIPLES: What Must This Page Do?

A visitor lands on the site. In the first 10 seconds they must answer:

| Second | Question | Answer |
|--------|----------|--------|
| 0-3 | What is this? | A solar workflow platform |
| 3-6 | Why should I care? | It eliminates manual mess in solar projects |
| 6-10 | Is it for me? | Yes — EPC, installer, homeowner, or O&M |

If ANY of these fail, they bounce.

The page must also handle 4 different visitor types, each with different needs:
- **EPC owner:** "Will this save me time and money?"
- **Installer:** "Will this make my site work easier?"
- **Homeowner:** "Should I get solar? Is it worth it?"
- **Investor/Partner:** "Is this company credible?"

---

## 🎨 THE NARRATIVE ARC

```
"You know solar is a mess. Here's proof. 
 Here's what we built to fix it. 
 Here's how it works — the magic part. 
 Here's what's inside. 
 Here's who uses it. 
 Here's proof it works. 
 Let's talk."
```

This is a **story**, not a feature list. Each section earns the next.

### S2: THE STORY — "Solar shouldn't be this hard."

**GOAL:** Answer "Why should I care?" Make them FEEL the pain.

**DESIGN THINKING:**
Forget the dark/light split. Forget even the side-by-side cards. Instead, think of this as an **editorial spread** — like something from a premium magazine or a Stripe blog post.

**VISUAL TREATMENT:**
- Background: Clean white
- A large, bold statement section that feels like a manifesto
- NOT a comparison grid. A narrative.

**LAYOUT:**
- Left column (60%): 
  - Small overline: "The problem"
  - Large headline: **"Solar shouldn't be this hard."** (5 words)
  - 2-3 punchy paragraphs, short sentences:
    "Today, solar projects run on WhatsApp groups, Excel sheets, and manual CAD. Leads get lost. Designs take days. Proposals have pricing errors. Installers get unclear instructions. Nobody has a single view of the project."
    "This isn't a technology problem. It's a connection problem."
  - CTA: "See how pvNXT fixes this" → scrolls to S4

- Right column (40%):
  - **ILLUSTRATION: The Mess → The Solution**
  ```
  [ILLUSTRATION: Solar Chaos to Clarity
  Size: 500x600px
  Style: Editorial line-art, teal + warm gray on white
  Left side (chaos): Scattered icons — phone (WhatsApp), spreadsheet, CAD cursor, map pin, email — overlapping, tangled with red X connections
  Right side (clarity): All same icons now connected in a clean circle around a central "pvNXT" hub with teal lines
  Transition: A subtle gradient from chaotic to organized, left to right
  Purpose: Show — don't tell — the transformation pvNXT enables]
  ```

**WHY THIS WORKS:** Instead of listing 7 problems (which feels like a checklist), we tell a story. The illustration does the heavy lifting. The emotional hook is: "You know this pain. We fixed it."

### S5: WHO IT'S FOR — "Built for every solar team."

**GOAL:** Answer "Is it for me?" for every visitor type.

**DESIGN THINKING:**
This is NOT another card grid. By now we've had enough cards (S4). Instead, this is a **role selector** — almost like a navigation aid. The user should self-identify.

**VISUAL TREATMENT:**
- Background: Subtle teal tint (bg-teal-50/50)
- 4 wide, horizontal lanes — one per role
- Each lane is clickable, linking to the relevant portal
- Think: not cards, but **destination signs**

**LAYOUT:**
A stacked set of 4 horizontal rows, each containing:

```
[ICON in circle]  ROLE NAME  —  "What you get"  →  [Explore Portal Name]
```

| Icon | Role | Subtitle | Link |
|------|------|----------|------|
| Building2 | **EPC Companies** | Design, propose, and track — in one workspace | Explore Studio → |
| Wrench | **Solar Installers** | Site tasks, photo proof, updates — from the rooftop | Explore Go → |
| Home | **Homeowners** | Feasibility, cost, tracking — no jargon | Explore Connect → |

- Lanes have generous vertical spacing

**WHY THIS WORKS:** No generic "Who Uses" cards. Each user immediately finds their lane and a clear next step.

## 📊 WHY THIS PLAN IS DIFFERENT FROM V1

| V1 | V2 |
|----|-----|
| Old Way as a comparison grid | Old Way as an **editorial narrative** with illustration |
| "The gap we close" headline | "Solar shouldn't be this hard." — more emotional |
| S4 had Atlas + timeline together | S3 is a **dedicated, immersive "magic" section** with 2 illustrations |
| Who Uses as icon cards | **Role selector lanes** — each user finds themselves |
| Results as stat cards | **Floating typography** — more confident, less "card" |
| Products before How It Works | **How It Works BEFORE Products** — show magic first, then what's inside |

**Key insight:** The narrative arc matters. You don't show products until you've made the person care. S2 makes them care. S3 wows them. THEN S4 shows products.

## 🎯 CONTENT TONE RULES

- Headlines: 3-5 words max. Punchy. No filler.
- Body: 2-3 lines max per paragraph
- No words like "seamless," "revolutionize," "empower," "cutting-edge"
- Sound like a real company, not a text generator
- Corporate but warm. Confident but not arrogant.

## ✅ VERDICT

This plan:
- Uses the terranxt design DNA (CSS, animations, fonts, colors)
